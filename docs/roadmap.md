## Design: Dynamic Context Injection (Layer 4)

> **Goal:** inject runtime-computed data into every agent's system prompt automatically, without requiring agents or leaders to explicitly request it.

### Concept

The current 3-layer system prompt will gain a **Layer 4 — Dynamic Context**, inserted between Layer 1 (engine context) and Layer 2 (agent role):

```
Layer 1 — ENGINE CONTEXT       (.ai-engine/engine_context.md — static)
Layer 4 — DYNAMIC CONTEXT      (computed at runtime, just before each LLM call)
Layer 2 — AGENT ROLE           (system_prompt.md — static)
Layer 3 — CURRENT TASK         (task_context.md — written by leader)
```

### Architecture

A **`DynamicContextProvider` interface** will be defined in a new `internal/context/` package:

```go
type DynamicContextProvider interface {
    Name() string
    Render(ctx context.Context, sb *sandbox.Sandbox) (string, error)
}
```

A **`DynamicContextRegistry`** holds a list of registered providers. At agent run time, the runner calls each provider in order, concatenates their output, and injects the result as Layer 4.

Adding a new dynamic context source = implementing the interface and registering it. No changes to the runner or other providers.

### Initial Providers

| Provider | Output | Notes |
|---|---|---|
| `WorkspaceTreeProvider` | Current file/directory tree of the workspace (excluding `.ai-engine/`) | Gives agents awareness of what already exists in the project |

### Future Providers (ideas)

| Provider | Output |
|---|---|
| `GitStatusProvider` | Output of `git status --short` — which files are modified/untracked |
| `RunningServicesProvider` | List of ports in use / processes running in the workspace |
| `RecentLogsProvider` | Last N lines of a relevant log file |
| `EnvSummaryProvider` | Relevant environment variables (e.g., `NODE_ENV`, `GO_VERSION`) |

### Configuration

Providers can be enabled/disabled per-agent or globally via `config.json`:

```json
{
  "dynamic_context": {
    "providers": ["workspace_tree"]
  }
}
```

If `dynamic_context` is omitted, all registered providers are enabled by default.
